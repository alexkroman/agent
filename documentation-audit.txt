CITATION 1: createServer() -- The Crown Jewel With No Crown
File: /home/user/agent/packages/aai/server.ts, line 58
Offense: The `createServer` function is THE primary public API for self-hosted
  deployments -- it is literally the `./server` subpath export -- and yet it
  has ZERO JSDoc. No @param, no @returns, no @example, no @public tag.

  - @example showing minimal usage: createServer({ agent }).listen(3000)
  - @public tag

*sets down red pen, removes glasses, pinches bridge of nose*

In summary: the public-facing types.ts file is a model citizen -- thorough
JSDoc, @public tags, code examples, linked references. The kv.ts, vector.ts,
and testing.ts files are also reasonably well documented. But beyond those
showcases, the documentation quality drops sharply. Internal modules that are
nonetheless exported (session.ts, direct-executor.ts, _session-otel.ts,
runtime.ts, s2s.ts) are documented at a level I would charitably describe as
"notes scribbled on a napkin."

The most urgent gaps are in security-critical code (_ssrf.ts), the primary
self-hosted API (server.ts createServer), and the complex session management
pipeline (session.ts, _session-otel.ts) where subtle generation-counter
logic and race-condition guards DEMAND clear explanatory documentation.
